# 產品路線圖

## 產品定位

Trend Radar 是一個協助內容創作者、品牌與行銷團隊整理熱門話題、判斷內容切入點並產生內容靈感的工具。

產品長期方向是從 Phase 1 Demo，逐步演進到可支援多資料來源、跨平台趨勢分析、AI 趨勢顧問與 SaaS 化的產品。

## 目前狀態

- Phase 1 Demo 已完成
- 目前進入 Week 11 展示準備階段
- Demo 可在 10 分鐘內完成展示
- Phase 1 主要依賴 mockData 與 JSON sample，不依賴臨場外部資料
- AI 目前為 mock / rule-based，尚未串接真實 AI API

## Phase 1：結訓展示版

目標：證明核心流程成立，讓使用者可以從固定展示資料看到熱門話題、話題詳情、AI mock 分析與文案生成。

核心流程：

```text
資料整合
→ 熱門話題分類
→ 話題詳情展示
→ AI mock 摘要分析
→ 爭議提醒
→ 內容靈感 / 文案生成
```

已完成功能：

- 匯入 mockData 或 JSON 資料
- Threads / Dcard 假資料展示
- Reddit 驗證流程
- Normalize 成統一資料格式
- 基本熱門分數計算
- Dashboard
- 所有熱門話題
- 符合產業熱門話題
- 自訂關鍵字熱門話題
- 話題卡片
- 話題詳情頁 / 彈窗
- 平台標籤
- Topic Tags
- 爭議提醒區
- AI 短摘要
- AI 完整摘要
- 內容靈感
- 文案生成
- 同一話題最新三版文案保留

Phase 1 限制：

- AI 仍為 mock / rule-based
- 尚未串接 OpenAI
- 尚未串接 Gemini
- 尚未完成正式前後端 API 串接
- PostgreSQL 尚未正式寫入
- 尚未完成 Phase 2 功能

## Phase 2：基礎產品化

目標：在不破壞 Phase 1 Demo 穩定性的前提下，補強低風險、高展示價值的產品功能。

候選功能：

- 真實 AI API
- 第二資料來源
- Source Analysis
- Dashboard 強化
- 使用者設定
- 收藏功能
- JSON / CSV 匯入強化
- 基礎 API 串接
- PostgreSQL 寫入與查詢

## Phase 3：趨勢分析強化

目標：從單純熱門分數進一步擴充成趨勢分析系統。

候選功能：

- Growth Rate
- Momentum 強化
- Lifecycle Stage
- 即將過氣判斷
- 趨勢生命週期
- 更完整的 TrendSignal
- 更完整的 TrendReport
- 多維度熱門分數

## Phase 4：跨平台與來源分析

目標：支援跨平台資料來源，判斷話題是否跨平台擴散。

候選功能：

- Cross Platform Signals
- Source Analysis
- 平台聲量比較
- 不同平台話題對照
- 跨平台爆紅判斷
- 平台差異洞察

## Phase 5：AI 趨勢顧問

目標：讓 AI 不只產生摘要與文案，也能提供趨勢觀察與策略建議。

候選功能：

- AI 推薦觀察
- AI 趨勢顧問聊天介面
- AI 內容策略建議
- AI 風險判斷強化
- AI 受眾洞察
- AI 產業切入建議

## Phase 6：SaaS 化

目標：將工具擴展為可長期使用的 SaaS 產品。

候選功能：

- 登入系統
- Workspace
- 使用者權限
- 團隊功能
- 付費會員
- API 使用量管理
- 自動寄送報告
- 長期資料保存
- 正式上線部署與維運

## 開發原則

- Phase 1 穩定性優先
- 不跳過 Phase 1 直接做 Phase 3 以後功能
- Phase 2 只能挑選低風險、高展示價值項目
- 不讓新功能破壞 Demo 主流程
- 展示資料必須可重複操作
