# 產品路線圖

本文件只維護產品 Phase 與能力演進方向，不維護目前狀態或細部開發待辦。

目前專案狀態請以 `docs/CURRENT_STATUS.md` 為準。開發待辦請以 `docs/開發路線圖.md` 為準。

## 產品定位

Trend Radar 是一個協助內容創作者、品牌與行銷團隊整理熱門話題、判斷內容切入點並產生內容靈感的工具。

長期方向是從 Demo 演進到可支援多資料來源、跨平台趨勢分析、AI 趨勢顧問與 SaaS 化的產品。

## Phase 1：Demo 核心流程

目標：

- 證明資料整合、熱門話題分類、話題詳情、AI 分析與文案生成的主流程成立。

產品能力：

- 固定展示資料。
- Topic Dashboard。
- Topic Detail。
- AI summary / warning / ideas / generated copy。
- 可重複展示的 Demo 主流程。

## Phase 2：基礎產品化

目標：

- 在 Demo 穩定性之上補強低風險、高展示價值的產品能力。

產品能力：

- AI provider adapter 與 fallback。
- PostgreSQL 基礎資料層。
- Dashboard API。
- User Settings。
- Favorites。
- 第二資料來源。
- Source Analysis。

## Phase 3：趨勢分析強化

目標：

- 從單純熱門分數擴充成更完整的趨勢分析系統。

產品能力：

- Growth Rate。
- Momentum。
- Lifecycle Stage。
- Trend Chart。
- 更完整的 TrendSignal / TrendReport。
- 多維度熱門分數。

## Phase 3 Lite / Extended：AI Copy Workflow

目標：

- 讓 AI 產生的文案從展示結果，演進成可編輯、可迭代、可管理的內容工作流。

產品能力：

- AI Copy Editor Modal。
- Tone Selector。
- Regenerate Copy。
- Multi-Version Output。

依賴方向：

- Copy Editor Modal 已正式採用為 Phase 3 Lite 下一個 MVP，是後續重新生成、多版本與歷史紀錄的基礎。
- Tone Selector 可先沿用既有語氣設定，再逐步擴充。
- Regenerate Copy 與 Multi-Version Output 需建立在穩定的 Modal 與文案狀態管理上。

## Phase 4：保存、管理與跨平台深化

目標：

- 支援長期使用情境，讓趨勢與文案結果可以被保存、回看與管理。

產品能力：

- Copy History。
- 更完整的跨平台來源分析。
- 長期資料保存策略。
- 使用者或 workspace 識別策略。

## Phase 5：輸出與 AI 趨勢顧問

目標：

- 讓內容結果可以進入外部工作流程，並讓 AI 提供更高層次的趨勢觀察與策略建議。

產品能力：

- Export：TXT / Markdown / DOCX / PDF。
- AI 趨勢顧問。
- AI 內容策略建議。
- AI 風險判斷強化。
- AI 受眾洞察。

## Phase 6：SaaS 化

目標：

- 將工具擴展為可長期使用的 SaaS 產品。

產品能力：

- Login。
- Workspace。
- Team collaboration。
- User permissions。
- Subscription。
- Usage management。
- Scheduled reports。
- 正式部署與長期維運。

## 產品原則

- Demo 穩定性優先。
- 不讓新功能破壞主流程。
- 外部服務必須有 fallback。
- 先完成最小閉環，再擴充管理、匯出與商業化能力。
