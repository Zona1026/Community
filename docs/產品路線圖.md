# 產品路線圖

## 文件目的

這份文件用來明確區分：

- 10 週結訓必須完成的功能
- 結訓後真正上線、產品化才需要做的功能

本專案可以有很大的發展空間，但職訓結訓專案最重要的是先完成一個可以完整展示的版本，證明核心流程成立。

核心流程：

```text
資料整合
→ 熱門話題分類
→ AI 摘要分析
→ 爭議提醒
→ 內容靈感 / 文案生成
```

因此，10 週內只專注 Phase 1。後續功能不刪掉，而是放到後面的 Phase。

## Phase 1：結訓展示版（10 週內必須完成）

### 目標

做出可以完整 Demo 的版本，證明本專案核心價值成立。

Phase 1 不追求產品完整上線，也不追求所有趨勢分析都做到很精細。重點是讓老師、同學或評審能在 10 分鐘內理解：

- 系統可以整合資料。
- 系統可以整理熱門話題。
- 系統可以用 AI 做摘要與風險提醒。
- 系統可以把熱門話題延伸成內容靈感或文案方向。

### 必須完成

- 匯入 mockData 或 JSON 資料
- 將不同平台資料 Normalize 成統一格式
- 支援 Threads 與 Dcard 的資料展示
- 熱門分數計算
- 首頁三個核心區塊：
  1. 所有熱門話題
  2. 符合產業熱門話題
  3. 自訂關鍵字熱門話題
- 話題卡片
- 話題詳情頁或詳情彈窗
- AI 短摘要
- AI 完整摘要
- 爭議提醒
- 語氣選擇
- 發文角度選擇
- AI 內容靈感或文案生成
- 最新三版文案保留
- 基本使用者設定頁
- 可完成 10 分鐘 Demo 展示

### 暫時不追求

- 真正全自動爬蟲
- 複雜趨勢生命週期
- 跨平台爆紅判斷
- 即將過氣判斷
- AI 推薦觀察
- 完整 SaaS 帳號權限

### Demo 展示重點

Phase 1 展示時建議強調：

1. 使用者完成基本設定。
2. 系統匯入 mockData 或 JSON。
3. 系統將 Threads 與 Dcard 資料整理成統一格式。
4. 首頁顯示三個熱門話題區塊。
5. 點擊話題查看詳情。
6. AI 產生短摘要與完整摘要。
7. 系統顯示爭議提醒。
8. 使用者選擇語氣與發文角度。
9. 系統產生內容靈感或文案。
10. 展示同一話題只保留最新三版。

## Phase 2：上線前版本

### 目標

讓系統從「可展示」變成「真的可以給少量使用者使用」。

Phase 2 開始處理比較實際的使用需求，例如登入、設定保存、資料匯入穩定性、錯誤處理與基本部署。

### 功能

- 使用者登入
- 使用者設定儲存
- 真實 JSON / CSV 匯入
- 第三方爬蟲工具資料串接，例如 Apify 或其他合法資料來源
- 手動重新匯入資料
- 每日排程匯入資料
- 今日熱門趨勢區塊
- 平台篩選
- 熱門標籤篩選
- 文案一鍵複製
- 收藏熱門話題
- 基本錯誤處理
- 基本部署上線

### 完成後的狀態

Phase 2 完成後，系統應該可以讓少量測試使用者使用，並能穩定完成資料匯入、熱門話題查看與 AI 分析流程。

## Phase 3：進階趨勢分析版

### 目標

讓系統不只顯示熱門，而是能判斷趨勢變化。

Phase 3 的重點是 Trend Engine，開始分析話題是否正在升溫、是否已經爆紅、是否正在衰退。

### 功能

- 快速升溫趨勢
- Growth Rate 成長率
- Momentum 趨勢動能
- Lifecycle Stage 趨勢生命週期：
  - 萌芽
  - 成長
  - 爆紅
  - 衰退
  - 過氣
- 趨勢歷史資料儲存
- 熱度變化圖表
- 即將過氣趨勢
- 不同時間窗比較
- 更完整的熱門分數公式

### 完成後的狀態

Phase 3 完成後，系統可以從「熱門列表工具」進一步變成「趨勢分析工具」。

使用者不只能看到什麼紅，還能知道：

- 它是不是正在升溫。
- 它是不是已經快退燒。
- 它適不適合現在跟進。

## Phase 4：跨平台與趨勢報告版

### 目標

讓系統可以比較不同平台的趨勢擴散狀況。

Phase 4 的重點是 Cross Platform Signals，讓使用者知道同一個話題在不同平台上的表現差異。

### 功能

- 跨平台爆紅趨勢
- Cross Platform Signals
- 比較 Threads、Dcard、PTT、Instagram、Facebook 等平台
- 判斷哪個平台最早升溫
- 判斷話題是否跨圈層擴散
- 趨勢報告頁
- 每日 / 每週趨勢報告
- 產業趨勢報告
- 匯出報告

### 完成後的狀態

Phase 4 完成後，系統可以協助使用者判斷：

- 話題是不是只在單一平台熱。
- 話題是否開始跨平台擴散。
- 哪個平台最適合優先觀察。
- 哪些趨勢值得放進週報或策略討論。

## Phase 5：AI 趨勢顧問版

### 目標

讓 AI 不只是摘要工具，而是能主動推薦值得觀察的趨勢。

Phase 5 的重點是 AI Insight，讓 AI 協助使用者判斷趨勢價值、切入風險與內容策略。

### 功能

- AI 推薦觀察
- AI 判斷低熱度但高潛力趨勢
- AI 推薦適合切入的產業
- AI 推薦不建議切入的話題
- AI 風險分級
- AI 提供內容策略建議
- AI 產生多種內容方向
- AI 趨勢助理聊天介面

### 完成後的狀態

Phase 5 完成後，系統可以從「趨勢分析工具」進一步變成「AI 趨勢顧問」。

使用者可以問：

- 今天有什麼值得觀察？
- 我的產業適合跟哪個趨勢？
- 哪些話題風險太高？
- 哪些趨勢現在跟已經太晚？
- 可以從這個趨勢延伸出哪些內容方向？

## Phase 6：商業化 SaaS 版本

### 目標

將系統發展成可長期營運的產品。

Phase 6 的重點是商業化、權限、團隊、訂閱與資料來源管理。

### 功能

- 多使用者帳號
- 工作區 Workspace
- 團隊協作
- 權限管理
- 訂閱制方案
- 付費會員功能
- 使用量限制
- API 使用量管理
- 多品牌管理
- 通知系統
- 自動寄送報告
- 更完整的資料來源管理

### 完成後的狀態

Phase 6 完成後，系統才算進入真正 SaaS 產品化階段。

這個階段會需要考慮：

- 收費模式
- 使用者權限
- 團隊協作
- 資料安全
- 系統穩定性
- API 成本
- 客戶支援

## 階段比較表

| Phase | 階段名稱 | 核心目標 | 是否屬於 10 週必做 |
| --- | --- | --- | --- |
| Phase 1 | 結訓展示版 | 證明核心流程成立 | 是 |
| Phase 2 | 上線前版本 | 讓少量使用者能實際使用 | 否 |
| Phase 3 | 進階趨勢分析版 | 判斷趨勢變化 | 否 |
| Phase 4 | 跨平台與趨勢報告版 | 比較平台擴散狀況 | 否 |
| Phase 5 | AI 趨勢顧問版 | 主動推薦與策略建議 | 否 |
| Phase 6 | 商業化 SaaS 版本 | 長期營運與收費 | 否 |

## 目前開發建議

目前仍以 Phase 1 為主要開發目標。

如果開發過程中出現新想法，例如：

- 做圖表
- 做趨勢生命週期
- 做 AI 聊天助理
- 做團隊帳號
- 做自動寄送報告

先放進 Phase 2 以後，不要在 Phase 1 尚未穩定完成前插入 10 週開發。

這樣可以避免專案範圍失控，也比較容易準時完成結訓展示。

如果 Phase 1 已經提前穩定完成，才可以依照 Phase 順序，從 Phase 2 挑選低風險、高展示價值的小功能提前加入。

## 開發原則

1. 10 週內主要目標是完成 Phase 1。
2. Phase 1 沒完成前，不開發 Phase 2 以後功能。
3. 任何新想法都先放進產品路線圖，不直接插入目前開發。
4. 結訓展示重點是證明核心流程成立，不是完成所有功能。
5. 未來上線需要 Phase 2；產品差異化主要來自 Phase 3 到 Phase 5。
6. Phase 1 提前穩定完成時，可以優先補 Phase 2 的低風險功能，但不跳到 Phase 3 以後。
