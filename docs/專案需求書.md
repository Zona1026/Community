# 專案需求書

## 專案定位

Trend Radar 是一個內容趨勢整理與靈感生成 Demo。Phase 1 目標是使用固定展示資料，驗證從資料整合、熱門話題分類、話題詳情、AI mock 分析到文案生成的主流程。

完整產品 Phase 規劃以 `docs/產品路線圖.md` 為準。本文件僅描述目前 Phase 1 Demo 與後續需求方向。

## 目前狀態

- Phase 1 Demo 已完成
- 已通過 Week 1~2、Week 3~4、Week 5~6、Week 7~8、Week 9~10 驗收
- 目前進入 Week 11 展示準備階段
- Demo 可在 10 分鐘內完成展示

## Phase 1 目標

Phase 1 要證明以下流程成立：

```text
資料整合
→ 熱門話題分類
→ 話題詳情展示
→ AI mock 摘要分析
→ 爭議提醒
→ 內容靈感 / 文案生成
```

## Phase 1 資料來源

Phase 1 主要使用固定展示資料：

- mockData
- Threads JSON sample
- Dcard JSON sample
- Reddit 驗證流程與 fallback data

Phase 1 不依賴臨場外部資料完成 Demo。

## Phase 1 必做功能

### 資料流程

- 匯入 mockData 或 JSON 資料
- 將 Threads 與 Dcard 資料轉成 `ContentItem`
- Normalize 成 `NormalizedContent`
- 建立 `TopicCluster`
- 建立 `TrendSignal`
- 建立 `TrendReport`
- 建立 `InspirationIdea`

### Dashboard

- 所有熱門話題
- 符合產業熱門話題
- 自訂關鍵字熱門話題
- 話題卡片
- Empty State

### 話題詳情

- Topic
- Score
- Momentum
- Summary
- Insight
- Source
- Content Count
- Related Content
- Inspiration Ideas
- Platform Tags
- Topic Tags
- 爭議提醒區

### AI Mock Analysis

- AI 短摘要
- AI 完整摘要
- 爭議提醒判斷
- 語氣選擇
- 發文角度選擇
- 內容靈感
- 文案生成
- 同一話題最新三版文案保留
- AI 失敗備援或錯誤提示

## 技術架構

前端：

- Next.js
- React
- TypeScript

後端：

- Flask
- `GET /health`
- JSON sample 載入功能

資料庫：

- PostgreSQL
- 連線統一使用 `DATABASE_URL`
- Phase 1 尚未正式寫入 PostgreSQL

AI：

- Phase 1 使用 mock / rule-based
- 尚未串接 OpenAI
- 尚未串接 Gemini

## Phase 1 已知限制

- 尚未串接正式前後端 API
- PostgreSQL 尚未正式使用
- AI 仍為 mock / rule-based
- 尚未完成 Phase 2 功能
- Reddit 即時資料可能受外部服務影響

## Phase 2 候選需求

Phase 2 可依產品路線圖順序挑選低風險、高展示價值功能：

- 真實 AI API
- 第二資料來源
- Source Analysis
- Dashboard 強化
- 使用者設定
- 收藏功能
