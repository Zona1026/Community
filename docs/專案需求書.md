# 專案需求書

本文件描述 Trend Radar 的產品需求與最終能力方向，不維護目前狀態，也不作為開發待辦清單。

目前狀態請以 `docs/CURRENT_STATUS.md` 為準。產品 Phase 請以 `docs/產品路線圖.md` 為準。開發待辦請以 `docs/開發路線圖.md` 為準。

## 專案定位

Trend Radar 是一個內容趨勢整理與靈感生成工具，協助內容創作者、品牌與行銷團隊從多來源內容中辨識熱門話題、理解來源脈絡，並產生可使用的內容靈感與文案。

## 目標使用者

- 內容創作者。
- 社群經營者。
- 品牌行銷團隊。
- 內容企劃與編輯。
- 需要快速掌握話題方向的小型團隊。

## 核心使用情境

使用者希望在短時間內完成：

1. 查看近期熱門話題。
2. 判斷話題是否符合自己的產業或關鍵字。
3. 了解話題來源、熱度與討論脈絡。
4. 查看 AI 產生的摘要、風險提醒與內容靈感。
5. 產生可編輯、可複製、可輸出的文案。

## 核心資料流程

```text
Source Data
-> ContentItem
-> NormalizedContent
-> TopicCluster
-> TrendSignal
-> TrendReport
-> InspirationIdea
-> Dashboard / Copy Workflow
```

## 主要功能需求

### 資料整合

- 匯入固定展示資料。
- 支援 JSON sample。
- 支援多來源內容轉換。
- 將來源資料 normalize 成統一格式。
- 保留外部資料不可用時的 fallback。

### Dashboard

- 顯示熱門話題。
- 支援產業與關鍵字篩選。
- 顯示趨勢總覽。
- 顯示來源分析。
- 顯示收藏內容。
- 提供 Empty State。

### 話題詳情

- 顯示 Topic。
- 顯示 Score。
- 顯示 Growth Rate。
- 顯示 Momentum。
- 顯示 Lifecycle Stage。
- 顯示 Summary / Insight。
- 顯示 Source / Content Count。
- 顯示 Related Content。
- 顯示 Inspiration Ideas。
- 顯示 Platform Tags / Topic Tags。
- 顯示爭議提醒。

### 趨勢分析

- 基本熱門分數。
- Growth Rate。
- Momentum。
- Lifecycle Stage。
- Score History。
- 未來可擴充 Trend Chart 與更完整生命週期分析。

### AI Analysis

- AI 短摘要。
- AI 完整摘要。
- 爭議提醒判斷。
- 內容靈感。
- 文案生成。
- AI provider 失敗時 fallback。

### AI Copy Workflow

- AI Copy Editor Modal。
- Editable textarea。
- Copy to Clipboard。
- Close modal。
- Tone Selector。
- Regenerate Copy。
- Multi-Version Output。
- Copy History。
- Export：TXT / Markdown / DOCX / PDF。

### 使用者設定

- 儲存產業類別。
- 儲存常用語氣。
- 儲存自訂關鍵字。
- 設定可影響 Dashboard 篩選與 AI 文案預設值。

### 收藏

- 收藏話題。
- 取消收藏。
- 顯示收藏清單。
- 外部 API 或資料庫不可用時保留 fallback。

## 技術需求

### Frontend

- 使用 Next.js / React / TypeScript。
- Dashboard 與 detail modal 必須可在 Demo 情境穩定操作。
- UI 應優先確保可讀性、可掃描性與主流程穩定。

### Backend

- 使用 Flask。
- 提供 health check。
- 提供 topics、user settings、favorites 等基礎 API。
- 外部資料或資料庫不可用時不應中斷 Demo。

### Database

- 使用 PostgreSQL 作為正式資料庫方向。
- 連線統一使用 `DATABASE_URL`。
- 不硬編碼帳密或供應商資訊。

### AI

- 支援 mock / rule-based fallback。
- 支援真實 AI provider adapter。
- 未設定 API key 或 provider 失敗時，不應阻斷主流程。

## 非目標

以下能力不作為早期 Demo 穩定性範圍：

- 完整登入與權限系統。
- Workspace 與團隊協作。
- 付費訂閱。
- 自動排程報告。
- 正式商業化部署與長期維運。
- 無 fallback 的即時外部資料依賴。
