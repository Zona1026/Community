# 專案需求書

## 專案名稱

AI 驅動的內容趨勢研究與靈感發想平台

## 文件職責

本文件負責描述：

- 專案背景
- 目標使用者
- 產品要解決的問題
- Phase 1 MVP 範圍
- 未來發展方向

功能所屬 Phase 以 `docs/產品路線圖.md` 為準。本需求書中的 MVP 範圍必須對齊產品路線圖的 Phase 1。

## 專案背景

社群小編每天需要追蹤大量社群平台內容，判斷哪些議題值得跟進、哪些內容適合品牌發文、哪些話題可能涉及爭議。若完全人工瀏覽 Threads、Dcard 等平台，容易耗費時間，也可能錯過熱門話題。

本專案希望建立一套可展示的自動化分析系統，協助社群小編快速掌握熱門內容，並透過 AI 產生摘要、爭議提醒與內容靈感或文案方向。

## 專案目標

Phase 1 的目標是完成結訓展示版，證明核心流程成立：

- 匯入 mockData 或 JSON 資料。
- 彙整 Threads 與 Dcard 的熱門話題。
- 將不同平台資料整理成統一格式。
- 依使用者設定的產業類別與自訂關鍵字進行分類篩選。
- 透過 AI 產生短摘要、完整摘要、爭議提醒與內容靈感或文案。
- 讓社群小編能快速找到值得跟進的熱門議題。

每日排程匯入、第三方資料工具串接與真實自動抓取屬於 Phase 2 以後，不列為 Phase 1 必做。

## 目標使用者

- 社群小編
- 公司行銷企劃
- 品牌經營者
- 自媒體創作者

## 第一版支援平台

- Threads
- Dcard

Phase 1 使用 mockData 或 JSON 展示 Threads 與 Dcard 資料，不做真正全自動爬蟲。

## 未來可擴充平台

這些平台屬於未來規劃，會依照產品路線圖放到 Phase 4 的跨平台與趨勢報告版處理。

## 首頁功能

Phase 1 首頁分成三個主要區塊。

### 1. 所有熱門話題

顯示系統匯入資料中的所有熱門內容，依熱門分數排序。

### 2. 符合產業熱門話題

依使用者設定的產業類別篩選熱門話題。

範例：

- 餐飲業
- 科技業
- 房地產
- 機車產業

### 3. 自訂關鍵字熱門話題

依使用者設定的關鍵字篩選熱門話題。

範例：

- AI
- 雅方
- iPhone
- 房價

## 熱門話題資料內容

每個熱門話題需包含以下資料：

- 標題
- 平台名稱
- 原始內容
- 原始連結
- 按讚數
- 留言數
- 熱門標籤
- 建立時間
- 匯入時間

### 熱門標籤範例

- 最多人討論
- 按讚數最高
- 留言數最多

## 熱門判斷方式

Phase 1 使用簡單計分方式，不導入複雜趨勢演算法。

熱門分數建議公式：

```text
熱門分數 = 按讚數 + 留言數 * 2
```

系統可依熱門分數排序，並取前 30 至 50 筆作為首頁展示資料。

Growth Rate、Momentum、Lifecycle Stage 等趨勢生命週期判斷屬於 Phase 3，不列為 Phase 1 必做。

## AI 分析功能

### 一句話短摘要

格式：

```text
一句話說明這個話題在討論什麼
```

用途：

- 讓使用者快速判斷是否需要深入查看。

### 完整摘要

內容需包含：

- 事件背景
- 大家在意什麼
- 適合怎麼發文或延伸內容

用途：

- 協助社群小編理解話題脈絡。
- 提供後續內容靈感或文案生成的基礎。

## 爭議提醒功能

如果話題涉及以下內容，需要顯示提醒：

- 食安問題
- 政治議題
- 名人爭議
- 社會事件

提醒文字：

```text
此話題可能涉及爭議，建議謹慎使用
```

Phase 1 可透過關鍵字比對加上 AI 判斷完成，不需建立複雜的內容審核系統。

## AI 內容靈感或文案生成功能

使用者可針對單一熱門話題產生內容靈感或社群文案方向。

### 可選語氣

- 激動
- 疑惑
- 專業
- 幽默
- 溫暖

### 發文角度

使用者可選擇：

- 跟風討論
- 專業分析
- 品牌觀點
- 幽默迷因

### Phase 2 未來規劃

- 自訂語氣儲存
- 文案一鍵複製
- 收藏熱門話題或文案

## 文案紀錄

每個熱門話題只保留最新三版文案。

範例：

- 版本 1
- 版本 2
- 版本 3

產生版本 4 時，系統自動刪除版本 1，只保留版本 2、版本 3、版本 4。

## Phase 1 MVP 功能範圍

### 必做功能

- 使用者基本設定頁
- 匯入 mockData 或 JSON 資料
- Threads 與 Dcard 假資料展示
- Normalize 成統一資料格式
- 熱門分數計算
- 熱門話題列表
- 三種首頁區塊
- 熱門話題詳細頁或詳情彈窗
- AI 短摘要
- AI 完整摘要
- 爭議提醒
- 語氣選擇
- 發文角度選擇
- AI 內容靈感或文案生成
- 文案版本保留最新三版

### Phase 2 未來規劃，可在 Phase 1 提前完成時選擇

- 文案一鍵複製
- 平台篩選
- 熱門標籤篩選
- 手動重新匯入資料
- 使用者設定儲存
- 收藏熱門話題
- 基本登入
- JSON / CSV 真實資料匯入
- 每日排程匯入資料

- Growth Rate
- Momentum
- Lifecycle Stage
- Cross Platform Signals
- AI 推薦觀察
- 趨勢報告
- 多人團隊協作
- 角色權限管理
- 社群平台自動發文
- 付費會員
- 複雜數據圖表
- 即時串流監控

## 成功展示標準

- 使用者可以完成基本設定。
- 系統可以匯入 mockData 或 JSON。
- 首頁可以看到三個熱門話題區塊。
- 每個熱門話題可以查看原始資料與 AI 分析結果。
- 使用者可以選擇語氣與發文角度產生內容靈感或文案。
- 同一話題只保留最新三版文案。
- Demo 不依賴臨場外部資料，也不依賴真正自動爬蟲。
