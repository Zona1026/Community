# 後端架構

本文件描述 Flask 後端架構，不維護目前狀態。

目前專案狀態請以 `docs/CURRENT_STATUS.md` 為準。技術選型請以 `docs/技術資源與選型.md` 為準。

## 技術架構

- Python。
- Flask。
- PostgreSQL。
- `DATABASE_URL`。

## 主要目錄

```text
backend/
├─ app.py
├─ db.py
├─ schema.sql
├─ topics.py
├─ requirements.txt
├─ .env.example
└─ README.md
```

## 主要路由

Health check：

```text
GET /health
```

Database：

```text
GET /db/status
POST /db/init
POST /db/smoke-test
```

Topics：

```text
GET /api/topics
GET /api/topics/<topic_id>
```

User Settings：

```text
GET /api/user-settings
PUT /api/user-settings
POST /api/user-settings
```

Favorites：

```text
GET /api/favorites
POST /api/favorites/<topic_id>
DELETE /api/favorites/<topic_id>
```

## Fallback 原則

- 未設定 `DATABASE_URL` 時，不應中斷 Demo 主流程。
- 資料庫不可用時，API 應回傳 fallback 或安全 skipped 結果。
- 外部服務不可用時，前端仍應可用固定資料展示。

## DATABASE_URL

```text
DATABASE_URL=postgresql://USER:PASSWORD@HOST:PORT/DATABASE
```

不得硬編碼資料庫帳密、Host 或供應商資訊。
