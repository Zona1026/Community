# 後端架構

## 技術選型

- Python Flask
- MySQL
- Flask-CORS
- MySQL Connector 或 SQLAlchemy
- APScheduler

第一版以單一 Flask 專案完成 API、資料抓取排程與 AI 分析流程，不拆成微服務。

## 後端目標

- 提供前端所需 API。
- 管理使用者設定。
- 每天固定時間抓取社群平台資料。
- 整理熱門話題並儲存至 MySQL。
- 呼叫 AI 產生摘要、爭議提醒與文案。
- 控制每個話題只保留最新三版文案。

## API 規劃

### 使用者設定

#### 取得設定

```text
GET /api/settings
```

回傳：

- 產業類別
- 常用語氣
- 每天資料抓取時間
- 自訂關鍵字
- 是否完成首次設定

#### 更新設定

```text
PUT /api/settings
```

用途：

- 儲存首次設定。
- 修改設定頁資料。

### 熱門話題

#### 取得首頁話題

```text
GET /api/topics/home
```

回傳三個區塊：

- 所有熱門話題
- 符合產業熱門話題
- 自訂關鍵字熱門話題

#### 取得單一話題詳情

```text
GET /api/topics/{topic_id}
```

回傳：

- 原始內容
- 熱門資料
- AI 短摘要
- AI 完整摘要
- 爭議提醒
- 最新三版文案

#### 手動觸發抓取

```text
POST /api/topics/crawl
```

用途：

- 展示時可手動更新資料。
- 模擬每日排程抓取效果。

### 語氣管理

#### 取得語氣清單

```text
GET /api/tones
```

包含預設語氣與使用者自訂語氣。

#### 新增自訂語氣

```text
POST /api/tones
```

欄位：

- 語氣名稱
- 語氣描述

#### 修改自訂語氣

```text
PUT /api/tones/{tone_id}
```

#### 刪除自訂語氣

```text
DELETE /api/tones/{tone_id}
```

### AI 文案

#### 產生文案

```text
POST /api/topics/{topic_id}/copies
```

欄位：

- 語氣
- 發文角度

流程：

1. 取得話題內容。
2. 取得使用者選擇的語氣與發文角度。
3. 呼叫 AI 產生社群文案。
4. 儲存文案版本。
5. 若超過三版，刪除最舊版本。
6. 回傳最新三版文案。

## 資料抓取流程

### 排程抓取

使用 APScheduler 依使用者設定時間每天執行。

第一版若多使用者處理過於複雜，可先以單一展示使用者為主。

流程：

1. 讀取使用者設定的抓取時間。
2. 到 Threads 與 Dcard 抓取近 24 小時資料。
3. 儲存原始話題資料。
4. 計算熱門分數。
5. 產生熱門標籤。
6. 呼叫 AI 產生摘要與爭議提醒。
7. 將分析結果存入資料庫。

### 手動抓取

展示時可提供手動按鈕呼叫後端 API，方便老師或同學立即看到流程。

## 平台抓取規劃

### Threads

第一版建議先採用可行資料來源或模擬資料展示。若正式 API 或抓取限制過高，可在文件與展示中說明限制。

抓取欄位：

- 標題或貼文摘要
- 原始內容
- 原始連結
- 按讚數
- 留言數
- 發文時間

### Dcard

可優先使用公開文章資料作為展示來源。

抓取欄位：

- 標題
- 文章內容
- 文章連結
- 愛心數
- 回覆數
- 發文時間

## AI 分析流程

### 短摘要

輸入：

- 標題
- 原始內容

輸出：

- 一句話短摘要

### 完整摘要

輸入：

- 標題
- 原始內容
- 留言或互動數據

輸出：

- 事件背景
- 大家在意什麼
- 適合怎麼發文

### 爭議提醒

判斷條件：

- 食安問題
- 政治議題
- 名人爭議
- 社會事件

輸出：

- 是否有爭議風險
- 爭議原因
- 提醒文字

## 熱門標籤產生規則

第一版可用簡單規則產生。

- 留言數最高：留言數位於前幾名
- 按讚數最高：按讚數位於前幾名
- 最多人討論：留言數高於設定門檻

## 錯誤處理

### API 錯誤

回傳一致格式：

```json
{
  "success": false,
  "message": "錯誤訊息"
}
```

### AI 失敗

若 AI 呼叫失敗：

- 保留原始話題資料。
- 摘要欄位顯示尚未產生。
- 前端可提供重新分析按鈕作為加分功能。

### 抓取失敗

若單一平台抓取失敗：

- 記錄錯誤。
- 不影響其他平台資料。
- 前端顯示最後成功更新時間。

## 安全與設定

敏感設定放在環境變數：

- MySQL 連線資訊
- AI API Key
- Flask Secret Key

第一版可先不做完整登入系統，以單一使用者設定為主。若時間允許，再加入簡單帳號登入。

## 驗收重點

- API 可正常供前端取得設定與熱門話題。
- 可手動觸發資料抓取。
- 可依排程時間執行抓取流程。
- AI 摘要與文案可正確儲存。
- 每個話題只保留最新三版文案。
- 發生抓取或 AI 錯誤時，系統不會整體中斷。
