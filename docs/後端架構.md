# 後端架構

## 技術選型

- Python Flask
- PostgreSQL
- pgvector
- Flask-CORS
- SQLAlchemy
- APScheduler

本專案後端定位為「趨勢資料整合、Trend Engine、AI 分析與內容靈感服務」。

後端不以完整爬蟲為核心，而是接收 mockData、JSON、CSV 或未來第三方工具資料，經過 Normalize、去重、向量化、群組化與趨勢計算後，提供前端查詢。

## 後端目標

- 匯入不同來源的內容資料。
- 將資料轉換為 NormalizedContent。
- 對內容去重與語意相似度處理。
- 建立 TopicCluster。
- 透過 Trend Engine 產生 TrendSignal。
- 產生 TrendReport。
- 根據趨勢產生 InspirationIdea。
- 提供前端趨勢研究 API。

## 資料整合流程

1. 匯入 `mockData.json`、JSON 或 CSV。
2. 儲存為 ContentItem，保留原始資料。
3. Normalize 成 NormalizedContent。
4. 執行去重。
5. 產生 embedding 並存入 pgvector。
6. Topic Clustering 形成 TopicCluster。
7. Trend Engine 計算 TrendSignal。
8. AI 產生 TrendReport。
9. 依使用者輸入產業或受眾產生 InspirationIdea。

## Trend Engine

Trend Engine 是核心服務層。

### Topic Clustering

將相似內容群組成 TopicCluster。

可使用：

- 關鍵字相似度
- 標題相似度
- 內容 embedding 相似度
- 平台來源與時間接近度

### Growth Rate

計算話題成長率。

範例：

```text
growth_rate = 最近時間窗互動數 / 前一時間窗互動數
```

若前一時間窗為 0，第一版可用簡化規則避免除以 0。

### Momentum

判斷話題動能。

狀態：

- rising
- stable
- cooling

### Heat Score

綜合計算趨勢熱度。

可納入：

- likes
- comments
- shares
- growth_rate
- platform_count
- recency
- exploding_content_count

### Lifecycle Stage

判斷趨勢生命週期。

階段：

- emerging：萌芽
- growing：成長
- viral：爆紅
- declining：衰退
- expired：過氣

### Cross Platform Signals

比較同一 TopicCluster 在不同平台的強度。

輸出：

- 各平台互動量
- 各平台成長率
- 哪個平台先升溫
- 是否跨平台擴散

### Exploding Content Detection

找出高互動、高成長、高分享的內容。

條件範例：

- 互動數高於同平台平均。
- 成長率高於門檻。
- 分享數或留言數明顯偏高。
- 在短時間內快速累積互動。

## AI 分析服務

AI 不直接替品牌發文，而是協助使用者理解趨勢與產生內容靈感。

AI 分析方向：

- 這個趨勢是什麼。
- 為什麼它正在紅。
- 適合哪些產業切入。
- 不適合哪些產業硬跟。
- 可以有哪些內容角度。
- 有哪些風險或過氣風險。
- 有哪些可轉化成品牌內容的靈感。

## API 規劃

### 匯入資料

```text
POST /api/import
```

用途：

- 匯入 mockData.json。
- 匯入第三方 JSON 或 CSV。
- 建立 ContentItem。

### 取得熱門趨勢列表

```text
GET /api/trends
```

查詢參數：

- platform
- industry
- lifecycle_stage
- min_heat_score
- keyword

### 查詢單一趨勢詳情

```text
GET /api/trends/{cluster_id}
```

回傳：

- TopicCluster
- TrendSignal
- TrendReport
- 代表內容
- 爆紅內容
- 跨平台訊號

### 依產業取得可借勢話題

```text
GET /api/trends/by-industry
```

查詢參數：

- industry
- target_audience
- risk_preference

### 根據某個趨勢產生內容靈感

```text
POST /api/trends/{cluster_id}/ideas
```

輸入：

- industry
- brand_type
- target_audience
- content_goal

輸出：

- InspirationIdea 清單

### 取得趨勢生命週期狀態

```text
GET /api/trends/{cluster_id}/lifecycle
```

### 取得跨平台趨勢比較

```text
GET /api/trends/{cluster_id}/platform-signals
```

## 服務層責任

### import_service

負責資料匯入。

### normalize_service

將不同平台資料轉成 NormalizedContent。

### dedupe_service

處理重複內容。

### embedding_service

產生內容向量，存入 pgvector。

### topic_clustering_service

建立 TopicCluster。

### trend_engine_service

計算 Growth Rate、Momentum、Heat Score、Lifecycle Stage。

### platform_signal_service

計算 Cross Platform Signals。

### ai_analysis_service

根據趨勢資料產生分析。

### inspiration_service

根據趨勢與使用者輸入條件產生 InspirationIdea。

### report_service

產生 TrendReport。

## SaaS 擴充方向

MVP 不優先做完整 multi-tenant。

保留未來擴充可能：

- users
- workspaces
- saved_trends
- saved_reports
- team_permissions

但第一版不把品牌資料庫或多租戶權限作為核心資料模型。

## 驗收重點

- 可匯入資料並 Normalize。
- 可產生 TopicCluster。
- 可計算 TrendSignal。
- 可查看熱門趨勢列表。
- 可查看單一趨勢詳情。
- 可依產業取得可借勢話題。
- 可產生內容靈感，而不是完整社群文案。
- 可說明 Trend Engine 的計算邏輯。
