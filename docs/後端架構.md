# 後端架構

## 文件職責

本文件負責描述資料流程、服務模組與系統架構。

後端架構可以保留未來功能構想，但必須標示所屬 Phase。Phase 2 以後功能不可寫成 Phase 1 必做。

功能所屬 Phase 以 `docs/產品路線圖.md` 為準。

## 技術選型

### Phase 1 必做

- Python Flask
- Flask-CORS
- SQLAlchemy
- MySQL

### Phase 2 未來規劃

- APScheduler
- 使用者登入相關套件
- JSON / CSV 真實資料匯入處理

### Phase 3 以後未來規劃

- PostgreSQL 或其他更適合分析的資料庫方案
- pgvector
- embedding 相似度處理
- 更完整的 Trend Engine

Phase 1 先以 MySQL 搭配簡單熱門分數完成展示，不導入 pgvector 與複雜向量分析。

## 後端定位

Phase 1 後端定位為：

```text
資料匯入
→ Normalize
→ 熱門分數計算
→ AI 摘要與爭議提醒
→ 內容靈感 / 文案生成
→ 提供前端查詢 API
```

後端不以完整爬蟲為核心。Phase 1 接收 mockData 或 JSON，經過 Normalize 與熱門分數計算後，提供前端查詢。

第三方爬蟲工具串接、每日排程匯入、跨平台分析、趨勢生命週期與 SaaS 權限都屬於後續 Phase。

## Phase 1 後端目標

- 匯入 mockData 或 JSON 資料。
- 將 Threads 與 Dcard 資料轉換為統一格式。
- 儲存原始內容與 Normalize 後內容。
- 使用簡單公式計算熱門分數。
- 提供所有熱門話題列表。
- 提供符合產業熱門話題列表。
- 提供自訂關鍵字熱門話題列表。
- 提供單一話題詳情。
- 產生 AI 短摘要。
- 產生 AI 完整摘要。
- 產生爭議提醒。
- 根據語氣與發文角度產生內容靈感或文案。
- 保留同一話題最新三版文案。

## Phase 1 資料整合流程

1. 匯入 `mockData.json` 或 JSON。
2. 儲存為 RawContent，保留原始資料。
3. Normalize 成 NormalizedContent。
4. 使用簡單公式計算熱門分數。
5. 依熱門分數排序形成 HotTopic。
6. 前端查詢熱門話題列表或詳情。
7. AI 依單一話題產生短摘要、完整摘要與爭議提醒。
8. AI 依使用者選擇的語氣與發文角度產生內容靈感或文案。
9. 同一話題只保留最新三版文案。

## Phase 1 熱門分數

Phase 1 使用簡單計分方式，不導入複雜趨勢演算法。

建議公式：

```text
熱門分數 = 按讚數 + 留言數 * 2
```

可依熱門分數排序，取前 30 至 50 筆作為首頁展示資料。

## Phase 1 API 規劃

### 匯入資料

```text
POST /api/import
```

用途：

- 匯入 mockData 或 JSON。
- 建立 RawContent。
- 執行 Normalize。
- 計算熱門分數。

### 取得所有熱門話題

```text
GET /api/topics/hot
```

查詢參數：

- keyword
- industry

### 取得符合產業熱門話題

```text
GET /api/topics/by-industry
```

查詢參數：

- industry

### 取得自訂關鍵字熱門話題

```text
GET /api/topics/by-keyword
```

查詢參數：

- keyword

### 查詢單一話題詳情

```text
GET /api/topics/{topic_id}
```

回傳：

- 原始內容
- Normalize 後內容
- 平台
- 熱門分數
- 熱門標籤
- AI 摘要
- 爭議提醒
- 最新三版文案

### 產生 AI 短摘要

```text
POST /api/topics/{topic_id}/short-summary
```

### 產生 AI 完整摘要

```text
POST /api/topics/{topic_id}/full-summary
```

### 產生爭議提醒

```text
POST /api/topics/{topic_id}/controversy-alert
```

### 產生內容靈感或文案

```text
POST /api/topics/{topic_id}/copies
```

輸入：

- tone
- angle
- industry

輸出：

- generated_text
- latest_versions

## Phase 1 服務層責任

### import_service

負責 mockData 或 JSON 匯入。

### normalize_service

將 Threads 與 Dcard 資料轉成統一格式。

### hot_score_service

計算 Phase 1 熱門分數。

### topic_service

提供熱門話題列表、產業篩選、關鍵字篩選與詳情查詢。

### ai_summary_service

根據話題資料產生短摘要與完整摘要。

### controversy_service

用關鍵字比對與 AI 判斷產生爭議提醒。

### copy_generation_service

根據話題、語氣與發文角度產生內容靈感或文案，並保留最新三版。

### user_setting_service

管理 Phase 1 基本設定。登入後持久化保存屬於 Phase 2。

- 文案一鍵複製

- 基本部署上線
