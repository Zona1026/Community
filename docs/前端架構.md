# 前端架構

## 文件職責

本文件負責描述畫面與互動設計。

前端架構可以保留未來功能構想，但必須標示所屬 Phase。Phase 2 以後功能不可寫成 Phase 1 必做。

功能所屬 Phase 以 `docs/產品路線圖.md` 為準。

## 產品定位

前端主頁應是「趨勢雷達 / Trend Radar」導向。

本產品不是單純的社群文案產生器，也不是讓使用者一進來輸入品牌並產生貼文。

主要流程：

1. 發現熱門話題
2. 理解話題內容
3. 評估是否適合自己的產業
4. 產生內容靈感或文案方向
5. 再由使用者決定是否延伸成實際貼文

## 技術選型

## Phase 1 話題卡片

每張卡片至少顯示：

- 話題名稱
- 一句話摘要或內容片段
- 熱門分數
- 主要來源平台，例如 Threads、Dcard
- 熱門標籤
- 風險提示，例如：可能有爭議、建議謹慎使用

卡片 CTA：

- 查看話題詳情
- 產生內容靈感或文案

### Phase 3 以後卡片欄位

以下欄位不列為 Phase 1 必做：

- Growth Rate
- Momentum
- Lifecycle Stage：萌芽 / 成長 / 爆紅 / 衰退 / 過氣
- 跨平台強度
- 過氣風險判斷

- 爭議提醒
- 語氣選擇
- 發文角度選擇

## Phase 1 內容靈感 / 文案產生區

這個功能協助使用者把熱門話題轉成內容方向或文案草稿。

表單欄位：

- 產業
- 常用語氣
- 發文角度

輸出內容：

- 可以怎麼切入這個話題
- 內容角度
- 文案草稿或開場方向
- 需要注意的風險

### Phase 2 以後未來規劃

- 文案一鍵複製
- 收藏文案或話題

## 視覺方向

整體風格應像「趨勢雷達 / 內容研究工具」，不是單純文案產生器。

Phase 1 建議使用：

- dashboard layout
- topic card
- score badge
- platform chips
- hot tag chips
- 風險提示
- 可點擊的詳情頁或詳情彈窗

Phase 3 以後再加入：

- lifecycle badge
- growth indicator
- trend chart

## 驗收重點

- 首頁第一眼是 Trend Radar，而不是文案工具。
- Dashboard 有三個 Phase 1 核心區塊。
- Topic Card 顯示熱門分數、平台、熱門標籤與風險提示。
- 使用者可以查看話題詳情。
- 話題詳情可以顯示 AI 摘要、爭議提醒與內容靈感或文案。
- 同一話題只保留最新三版文案。
- Phase 2 以後功能有保留規劃，但不列為 Phase 1 必做。
