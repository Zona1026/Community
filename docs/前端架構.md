# 前端架構

## 技術選型

- Vue 3
- Bootstrap
- Vue Router
- Axios

第一版以前後端分離方式開發。前端負責畫面呈現、表單操作、呼叫 API 與狀態顯示，不在前端處理複雜資料分析。

## 前端目標

- 提供社群小編容易理解的操作流程。
- 首頁快速呈現熱門話題。
- 讓使用者可以查看 AI 分析結果。
- 讓使用者可以產生並管理文案版本。
- 介面保持簡潔，適合職訓班三個月內完成。

## 頁面規劃

### 1. 首次設定頁

路徑建議：

```text
/setup
```

功能：

- 設定產業類別。
- 設定常用語氣。
- 設定每天資料抓取時間。
- 設定自訂關鍵字。

表單欄位：

- 產業類別：下拉選單
- 常用語氣：下拉選單
- 每天資料抓取時間：時間選擇
- 自訂關鍵字：文字輸入，可用逗號分隔

送出後導向首頁。

### 2. 首頁

路徑建議：

```text
/
```

首頁分成三個區塊：

- 所有熱門話題
- 符合產業熱門話題
- 自訂關鍵字熱門話題

每個區塊使用列表或卡片呈現熱門話題。

每筆熱門話題顯示：

- 標題
- 平台名稱
- 按讚數
- 留言數
- 熱門標籤
- 一句話短摘要

操作：

- 查看詳情
- 重新整理資料

### 3. 熱門話題詳細頁

路徑建議：

```text
/topics/:id
```

顯示內容：

- 標題
- 平台名稱
- 原始內容
- 原始連結
- 按讚數
- 留言數
- 熱門標籤
- 一句話短摘要
- 完整摘要
- 爭議提醒

操作：

- 選擇文案語氣
- 選擇發文角度
- 產生 AI 文案
- 查看最新三版文案

### 4. 語氣管理頁

路徑建議：

```text
/tones
```

功能：

- 查看預設語氣。
- 新增自訂語氣。
- 編輯自訂語氣。
- 刪除自訂語氣。

預設語氣：

- 激動
- 疑惑
- 專業
- 幽默
- 溫暖

自訂語氣欄位：

- 語氣名稱
- 語氣描述

### 5. 設定頁

路徑建議：

```text
/settings
```

功能：

- 修改產業類別。
- 修改常用語氣。
- 修改每天資料抓取時間。
- 修改自訂關鍵字。

## 元件規劃

### Layout

共用版型，包含：

- 導覽列
- 主要內容區

### TopicList

熱門話題列表元件。

Props：

- 標題
- 話題陣列
- 載入狀態

### TopicCard

單一熱門話題卡片。

顯示：

- 標題
- 平台
- 按讚數
- 留言數
- 標籤
- 短摘要

### TopicDetail

熱門話題詳細內容元件。

### CopyGenerator

文案生成元件。

包含：

- 語氣選擇
- 發文角度選擇
- 生成按鈕
- 生成中狀態

### CopyVersionList

文案版本列表元件。

只顯示最新三版。

### AlertBadge

爭議提醒元件。

當話題涉及爭議時顯示：

```text
此話題可能涉及爭議，建議謹慎使用
```

## API 串接規劃

前端透過 Axios 呼叫後端 API。

主要 API：

- 取得使用者設定
- 更新使用者設定
- 取得首頁熱門話題
- 取得單一話題詳情
- 產生 AI 文案
- 取得文案版本
- 新增自訂語氣
- 修改自訂語氣
- 刪除自訂語氣

## 狀態管理

第一版可先使用 Vue 內建狀態與元件 props，不一定需要 Pinia。

若後續頁面狀態變複雜，再加入 Pinia 管理：

- 使用者設定
- 語氣清單
- 首頁篩選條件

## 介面風格

- 使用 Bootstrap 建立一致的表單、按鈕與卡片。
- 首頁以資訊清楚為主，不做過度動畫。
- 使用不同 badge 顯示平台與熱門標籤。
- 爭議提醒使用醒目但不誇張的警示樣式。

## 前端資料流程

1. 使用者進入系統。
2. 前端檢查是否已完成首次設定。
3. 未完成設定時導向 `/setup`。
4. 已完成設定時進入首頁。
5. 首頁呼叫 API 取得三個區塊資料。
6. 使用者點擊話題進入詳情頁。
7. 詳情頁呼叫 API 取得原始資料、摘要與文案版本。
8. 使用者選擇語氣與角度，呼叫 API 產生文案。
9. 前端重新取得最新三版文案並更新畫面。

## 驗收重點

- 首次設定流程可正常完成。
- 首頁三個區塊資料清楚。
- 詳細頁能呈現 AI 分析結果。
- 文案生成操作直覺。
- 最新三版文案顯示正確。
- 手機與桌機畫面都可正常瀏覽。
