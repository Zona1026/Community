# 資料庫設計

## 資料庫選型

本專案主資料庫統一使用 PostgreSQL。

開發階段若已使用 Aiven PostgreSQL，可暫時保留。免費部署階段改用 Neon PostgreSQL Free。所有資料庫連線統一透過 `DATABASE_URL` 管理，避免程式碼綁死 Aiven、Neon 或任何單一資料庫供應商。

未來進階分析可搭配 pgvector。

原因：

- PostgreSQL 適合結構化資料。
- pgvector 可在 Phase 3 以後支援 embedding 相似度查詢。
- Topic Clustering 未來可使用向量相似度輔助。
- 未來若擴充 SaaS，PostgreSQL 也較容易延伸。

## 核心資料模型

資料模型不以 brand 為核心，而是以內容、話題、趨勢訊號與靈感為核心。

至少包含：

- ContentItem
- NormalizedContent
- TopicCluster
- TrendSignal
- TrendReport
- InspirationIdea

## content_items

對應 ContentItem。

存放匯入時的單筆原始內容。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | BIGSERIAL PK | 原始內容 ID |
| source_name | VARCHAR(100) | 資料來源，例如 mockData、Apify、n8n |
| platform_name | VARCHAR(100) | 平台名稱 |
| platform_code | VARCHAR(50) | 平台代碼 |
| external_content_id | VARCHAR(255) | 外部平台內容 ID |
| raw_data | JSONB | 原始資料 |
| imported_at | TIMESTAMP | 匯入時間 |
| status | VARCHAR(50) | imported、normalized、failed |
| error_message | TEXT | 錯誤訊息 |

## normalized_contents

對應 NormalizedContent。

存放標準化後內容，是前端、Trend Engine 與 AI 分析的主要資料來源。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | BIGSERIAL PK | 標準化內容 ID |
| content_item_id | BIGINT FK | 對應 content_items.id |
| platform_name | VARCHAR(100) | 平台名稱 |
| platform_code | VARCHAR(50) | 平台代碼 |
| external_content_id | VARCHAR(255) | 外部內容 ID |
| title | VARCHAR(255) | 標題 |
| body | TEXT | 內容本文 |
| url | TEXT | 原始連結 |
| author_name | VARCHAR(255) | 作者名稱，可為空 |
| content_type | VARCHAR(100) | 內容形式，例如 post、comment、image、video、thread |
| likes | INT | 按讚數 |
| comments | INT | 留言數 |
| shares | INT | 分享數 |
| published_at | TIMESTAMP | 發布時間 |
| collected_at | TIMESTAMP | 收集時間 |
| trend_score | NUMERIC | 單篇內容初始熱度 |
| extra_data | JSONB | 額外資料 |
| embedding | VECTOR | 內容向量，使用 pgvector |
| created_at | TIMESTAMP | 建立時間 |
| updated_at | TIMESTAMP | 更新時間 |

## topic_clusters

對應 TopicCluster。

代表一組相似內容形成的話題群組。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | BIGSERIAL PK | 話題群組 ID |
| cluster_title | VARCHAR(255) | 話題名稱 |
| cluster_summary | TEXT | 話題摘要 |
| representative_content_id | BIGINT FK | 代表內容 |
| keywords | JSONB | 關鍵字 |
| platforms | JSONB | 涵蓋平台 |
| content_count | INT | 群組內內容數 |
| first_seen_at | TIMESTAMP | 首次出現時間 |
| last_seen_at | TIMESTAMP | 最近出現時間 |
| cluster_embedding | VECTOR | 群組向量 |
| created_at | TIMESTAMP | 建立時間 |
| updated_at | TIMESTAMP | 更新時間 |

## topic_cluster_items

TopicCluster 與 NormalizedContent 的關聯表。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | BIGSERIAL PK | 關聯 ID |
| topic_cluster_id | BIGINT FK | topic_clusters.id |
| normalized_content_id | BIGINT FK | normalized_contents.id |
| similarity_score | NUMERIC | 相似度分數 |
| is_representative | BOOLEAN | 是否為代表內容 |
| created_at | TIMESTAMP | 建立時間 |

## trend_signals

對應 TrendSignal。

存放 Trend Engine 計算出的趨勢訊號。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | BIGSERIAL PK | 訊號 ID |
| topic_cluster_id | BIGINT FK | topic_clusters.id |
| heat_score | NUMERIC | 趨勢熱度 |
| growth_rate | NUMERIC | 成長率 |
| momentum | VARCHAR(50) | rising、stable、cooling |
| lifecycle_stage | VARCHAR(50) | emerging、growing、viral、declining、expired |
| platform_count | INT | 涵蓋平台數 |
| total_likes | INT | 總按讚數 |
| total_comments | INT | 總留言數 |
| total_shares | INT | 總分享數 |
| exploding_content_count | INT | 爆紅內容數 |
| calculated_at | TIMESTAMP | 計算時間 |

| 欄位 | 型別 | 說明 |

| idea_summary | TEXT | 靈感摘要 |
| content_angle | VARCHAR(255) | 內容角度 |
| suggested_format | VARCHAR(100) | 建議內容形式 |
| trend_connection | TEXT | 與趨勢的關聯 |
| risk_note | TEXT | 風險提醒 |
| stale_risk | VARCHAR(50) | 過氣風險 |
| created_at | TIMESTAMP | 建立時間 |

## MVP 簡化建議

- 不建立品牌資料庫。
- 不做完整 multi-tenant。
- 使用 mockData.json 驗證資料流程。
- Topic Clustering 可先用關鍵字與簡單向量相似度。
- Heat Score 可先用簡化公式。
- TrendReport 與 InspirationIdea 可先按需產生並存入資料庫。
