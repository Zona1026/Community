# 資料庫設計

## 資料庫

資料庫使用 MySQL。

第一版以單一使用者展示為主，但資料表保留 `user_id` 欄位，方便未來擴充多使用者。

## 資料表總覽

- users
- user_settings
- user_keywords
- tones
- topics
- topic_tags
- topic_ai_analysis
- copy_versions
- crawl_logs

## users

使用者資料表。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | 使用者 ID |
| name | VARCHAR(100) | 使用者名稱 |
| email | VARCHAR(255) | Email |
| created_at | DATETIME | 建立時間 |
| updated_at | DATETIME | 更新時間 |

第一版若不做登入，可建立一筆預設使用者資料。

## user_settings

使用者設定資料表。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | 設定 ID |
| user_id | INT | 使用者 ID |
| industry_category | VARCHAR(100) | 產業類別 |
| default_tone | VARCHAR(100) | 常用語氣 |
| daily_crawl_time | TIME | 每天資料抓取時間 |
| is_setup_completed | BOOLEAN | 是否完成首次設定 |
| created_at | DATETIME | 建立時間 |
| updated_at | DATETIME | 更新時間 |

## user_keywords

使用者自訂關鍵字資料表。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | 關鍵字 ID |
| user_id | INT | 使用者 ID |
| keyword | VARCHAR(100) | 自訂關鍵字 |
| created_at | DATETIME | 建立時間 |

範例：

- AI
- 雅方
- iPhone
- 房價

## tones

語氣資料表。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | 語氣 ID |
| user_id | INT NULL | 使用者 ID，預設語氣可為 NULL |
| name | VARCHAR(100) | 語氣名稱 |
| description | TEXT | 語氣描述 |
| is_default | BOOLEAN | 是否為系統預設語氣 |
| created_at | DATETIME | 建立時間 |
| updated_at | DATETIME | 更新時間 |

預設資料：

| name | description |
| --- | --- |
| 激動 | 用高情緒、強烈感受的方式撰寫 |
| 疑惑 | 用提問、好奇、不確定感的方式撰寫 |
| 專業 | 用理性分析、可信任的方式撰寫 |
| 幽默 | 用輕鬆有趣的方式撰寫 |
| 溫暖 | 用親切、理解、陪伴感的方式撰寫 |

自訂範例：

| name | description |
| --- | --- |
| 酸民風 | 用吐槽但不失禮貌的方式撰寫 |

## topics

熱門話題資料表。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | 話題 ID |
| platform | VARCHAR(50) | 平台名稱，例如 Threads、Dcard |
| title | VARCHAR(255) | 標題 |
| original_content | TEXT | 原始內容 |
| original_url | VARCHAR(500) | 原始連結 |
| like_count | INT | 按讚數 |
| comment_count | INT | 留言數 |
| hot_score | INT | 熱門分數 |
| posted_at | DATETIME | 原文發布時間 |
| crawled_at | DATETIME | 抓取時間 |
| created_at | DATETIME | 建立時間 |
| updated_at | DATETIME | 更新時間 |

熱門分數：

```text
hot_score = like_count + comment_count * 2
```

## topic_tags

熱門標籤資料表。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | 標籤 ID |
| topic_id | INT | 話題 ID |
| tag_name | VARCHAR(100) | 標籤名稱 |
| created_at | DATETIME | 建立時間 |

標籤範例：

- 最多人討論
- 按讚數最高
- 留言數最多

## topic_ai_analysis

AI 分析結果資料表。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | 分析 ID |
| topic_id | INT | 話題 ID |
| short_summary | VARCHAR(255) | 一句話短摘要 |
| full_summary | TEXT | 完整摘要 |
| is_controversial | BOOLEAN | 是否涉及爭議 |
| controversy_reason | TEXT | 爭議原因 |
| controversy_warning | VARCHAR(255) | 爭議提醒文字 |
| created_at | DATETIME | 建立時間 |
| updated_at | DATETIME | 更新時間 |

爭議提醒文字：

```text
此話題可能涉及爭議，建議謹慎使用
```

## copy_versions

AI 文案版本資料表。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | 文案 ID |
| topic_id | INT | 話題 ID |
| user_id | INT | 使用者 ID |
| version_number | INT | 版本編號 |
| tone_name | VARCHAR(100) | 語氣名稱 |
| angle | VARCHAR(100) | 發文角度 |
| content | TEXT | 文案內容 |
| created_at | DATETIME | 建立時間 |

發文角度：

- 跟風討論
- 專業分析
- 品牌觀點
- 幽默迷因

版本規則：

- 每個 `topic_id` 加 `user_id` 最多保留三筆。
- 新增第四筆時，刪除最舊的一筆。
- `version_number` 可持續遞增，不需要重排。

## crawl_logs

資料抓取紀錄表。

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | INT PK AUTO_INCREMENT | 紀錄 ID |
| platform | VARCHAR(50) | 平台名稱 |
| status | VARCHAR(50) | 狀態，例如 success、failed |
| message | TEXT | 訊息或錯誤內容 |
| crawled_count | INT | 抓取筆數 |
| started_at | DATETIME | 開始時間 |
| finished_at | DATETIME | 結束時間 |

用途：

- 顯示最後更新時間。
- Debug 抓取流程。
- 展示排程執行紀錄。

## 關聯說明

- users 1 對 1 user_settings
- users 1 對多 user_keywords
- users 1 對多 copy_versions
- users 1 對多 tones
- topics 1 對多 topic_tags
- topics 1 對 1 topic_ai_analysis
- topics 1 對多 copy_versions

## 索引建議

為了讓首頁查詢更順暢，建議建立索引：

- topics.platform
- topics.hot_score
- topics.crawled_at
- topics.posted_at
- topic_tags.topic_id
- topic_tags.tag_name
- copy_versions.topic_id
- copy_versions.user_id

## 第一版簡化建議

若開發時間不足，可簡化：

- 不做 users 登入，只保留一位預設使用者。
- topic_tags 可先用一張表儲存，也可先在 topics 加上簡單文字欄位。
- crawl_logs 可先只記錄成功與失敗，不做詳細監控。
- AI 分析結果可在抓取完成後批次產生，不需要即時產生。
