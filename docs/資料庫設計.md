# 資料庫設計

本文件描述資料庫設計方向與 schema 候選，不維護目前狀態。

目前專案狀態請以 `docs/CURRENT_STATUS.md` 為準。技術選型請以 `docs/技術資源與選型.md` 為準。

## 設計原則

- 使用 PostgreSQL 作為正式資料庫。
- 使用 `DATABASE_URL` 管理連線。
- 不硬編碼帳密或供應商資訊。
- 外部資料庫不可用時，Demo 應保留 fallback。
- 未來若需要 embedding 或向量搜尋，再評估 pgvector。

## 目前最小資料表

後端 schema 位於：

- `backend/schema.sql`

目前最小資料表：

- `topics`
- `user_settings`
- `generated_copies`
- `favorites`

## 資料流程對照

前端目前主要資料型別：

- `ContentItem`
- `NormalizedContent`
- `TopicCluster`
- `TrendSignal`
- `TrendReport`
- `InspirationIdea`
- `AiAnalysis`

資料流：

```text
JSON / MockData / Reddit
↓
ContentItem
↓
NormalizedContent
↓
TopicCluster
↓
TrendSignal
↓
TrendReport
↓
InspirationIdea
↓
Dashboard UI
```

- id
- platform

- content
- url
- like_count

- hot_tags
- industry_tags
- keywords
- created_at
- imported_at
- raw_json

候選欄位：

- id
- content_item_id

- topic_title
- normalized_text
- original_url
- engagement_score
- hot_score

- like_count
- comment_count

- keywords

候選欄位：
