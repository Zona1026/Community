# 技術使用清單

本文件用來記錄本專案使用到的技術、工具，以及它們在專案中的用途。

這份文件不是技術決策紀錄，因此不說明為什麼選擇某項技術，也不比較放棄方案。

## 一、前端技術

### Vue 3

- 用途：建立前端單頁式應用程式
- 使用位置：`frontend/`
- 對應功能：
  - 趨勢雷達首頁
  - 熱門話題卡片
  - 話題詳情頁或詳情彈窗
  - AI 分析結果顯示
  - 文案生成互動區

### Bootstrap

- 用途：快速建立響應式版面與 UI 元件
- 使用位置：`frontend/`
- 對應功能：
  - 導覽列
  - 按鈕
  - 卡片
  - 表單
  - Badge 標籤
  - Modal 彈窗

### Vue Router

- 用途：管理前端頁面路由
- 使用位置：`frontend/`
- 對應頁面：
  - 首頁 Dashboard
  - 話題詳情頁
  - 使用者設定頁

### Axios

- 用途：前端呼叫後端 API
- 使用位置：`frontend/`
- 對應功能：
  - 取得熱門話題列表
  - 取得話題詳情
  - 送出 AI 文案生成請求
  - 取得文案歷史版本

## 二、後端技術

### Python

- 用途：後端主要開發語言
- 使用位置：`backend/`
- 對應功能：
  - 資料匯入流程
  - Normalize 資料標準化
  - 熱門分數計算
  - AI 功能串接
  - REST API 服務邏輯

### Flask

- 用途：建立 REST API 服務
- 使用位置：`backend/`
- 對應功能：
  - 熱門話題 API
  - 話題詳情 API
  - AI 摘要 API
  - AI 文案生成 API
  - 使用者設定 API

### Flask-CORS

- 用途：允許前端 Vue 呼叫後端 Flask API
- 使用位置：`backend/`
- 對應功能：
  - 前端呼叫熱門話題 API
  - 前端呼叫話題詳情 API
  - 前端呼叫 AI 摘要與文案生成 API
  - 處理前後端分離開發時的跨來源請求

### SQLAlchemy

- 用途：操作 PostgreSQL 資料庫
- 使用位置：`backend/`
- 對應功能：
  - 建立資料模型
  - 新增、查詢、更新、刪除資料
  - 操作使用者設定
  - 操作熱門話題資料
  - 操作文案生成紀錄

## 三、資料庫技術

### PostgreSQL

- 用途：儲存系統主要資料
- 使用位置：資料庫伺服器或雲端 PostgreSQL 服務
- 對應資料：
  - 使用者設定
  - 原始匯入資料
  - 標準化後資料
  - 熱門話題
  - AI 摘要結果
  - AI 文案生成紀錄
  - 最新三版文案
- 連線方式：後端統一讀取 `DATABASE_URL`
- 開發階段：若已使用 Aiven PostgreSQL，可暫時保留
- 免費部署階段：改用 Neon PostgreSQL Free

## 四、AI 技術

### OpenAI API

- 用途：提供 AI 摘要與內容生成能力
- 使用位置：`backend/`
- 對應功能：
  - 一句話短摘要
  - 完整摘要
  - 爭議提醒判斷
  - 內容靈感產生
  - 社群文案生成

### Prompt 檔案

- 用途：管理不同 AI 功能使用的提示詞
- 使用位置：`backend/prompts/`
- 對應檔案：
  - `summary_prompt.md`
  - `risk_prompt.md`
  - `copywriting_prompt.md`
- 對應功能：
  - 控制摘要輸出格式
  - 控制爭議提醒判斷方式
  - 控制文案生成語氣與角度

## 五、資料整合技術

### mockData.json

- 用途：Phase 1 開發與展示用資料來源
- 使用位置：
  - `frontend/src/data/`
  - 或 `backend/data/`
- 對應功能：
  - 模擬 Threads 與 Dcard 資料
  - 展示資料匯入流程
  - 展示熱門話題列表
  - 避免 Demo 依賴臨場外部資料

### JSON 匯入

- 用途：匯入第三方工具或手動整理的資料
- 使用位置：`backend/`
- 對應功能：
  - 匯入社群平台資料
  - 匯入展示用資料
  - 匯入未來第三方工具產生的資料

### Normalize 資料標準化

- 用途：將不同平台資料轉換成統一格式
- 使用位置：`backend/`
- 對應欄位：
  - `platform`
  - `title`
  - `body`
  - `url`
  - `likes`
  - `comments`
  - `shares`
  - `published_at`
  - `collected_at`
  - `extra_data`
- 對應功能：
  - 統一 Threads 與 Dcard 資料格式
  - 讓熱門分數可以用同一套欄位計算
  - 讓前端可以用一致格式顯示資料

## 七、開發工具

### VS Code

- 用途：主要程式碼編輯器
- 使用位置：本機開發環境
- 對應工作：
  - 編輯前端程式碼
  - 編輯後端程式碼
  - 編輯專案文件
  - 管理專案資料夾

### Git

- 用途：版本控制
- 使用位置：本機開發環境
- 對應工作：
  - 追蹤檔案修改
  - 建立提交紀錄
  - 管理分支
  - 回溯開發歷程

### GitHub

- 用途：程式碼備份與版本管理
- 使用位置：遠端程式碼倉庫
- 對應工作：
  - 備份專案程式碼
  - 管理版本
  - 未來可用於協作與部署串接

### npm

- 用途：管理前端套件
- 使用位置：`frontend/`
- 對應工作：
  - 安裝前端套件
  - 執行前端開發伺服器
  - 執行前端建置指令

### pip

- 用途：管理 Python 後端套件
- 使用位置：`backend/`
- 對應工作：
  - 安裝 Flask
  - 安裝 Flask-CORS
  - 安裝 SQLAlchemy
  - 安裝 OpenAI API 相關套件
