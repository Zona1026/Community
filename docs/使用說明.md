# 使用說明

## 這份文件是給誰看的

這份文件是給剛開始實作專案的開發者使用，目標是讓你知道：

- 專案資料夾在哪裡。
- 前端怎麼啟動。
- 後端未來怎麼啟動。
- 資料庫未來怎麼準備。
- 常見錯誤怎麼處理。

目前專案仍在規劃與前端骨架階段，後端程式尚未建立完成。

## 專案定位提醒

本專案不是社群文案產生器。

本專案是：

```text
AI 驅動的內容趨勢研究與靈感發想平台
```

主要流程是：

1. 發現趨勢
2. 理解趨勢
3. 評估是否適合自己的產業
4. 產生內容靈感
5. 再由使用者決定是否延伸成實際貼文

## 需要先安裝的工具

### 1. Git

用來管理版本與推上 GitHub。

確認是否安裝：

```powershell
git --version
```

如果看得到版本號，代表 Git 可以使用。

### 2. Node.js

用來啟動前端 Vue 專案。

確認是否安裝：

```powershell
node -v
npm -v
```

建議安裝 Node.js LTS 版本。

### 3. Python

未來後端 Flask 會使用 Python。

確認是否安裝：

```powershell
python --version
```

### 4. PostgreSQL

未來資料庫會使用 PostgreSQL，並搭配 pgvector。

MVP 前期如果只做前端展示，可以先不用安裝資料庫。

## 如何啟動前端

請先進入前端資料夾：

```powershell
cd frontend
```

第一次啟動前，需要安裝套件：

```powershell
npm install
```

安裝完成後，啟動前端開發伺服器：

```powershell
npm run dev
```

看到類似以下訊息時，代表啟動成功：

```text
Local: http://localhost:5173/
```

接著用瀏覽器打開：

```text
http://localhost:5173/
```

## 如何停止前端

在正在執行 `npm run dev` 的終端機中按：

```text
Ctrl + C
```

如果出現是否終止的問題，輸入：

```text
Y
```

## 未來如何啟動後端

目前後端尚未實作。

未來後端建立後，預計資料夾會長這樣：

```text
backend/
├── app.py
├── requirements.txt
├── routes/
├── services/
└── repositories/
```

預計啟動流程如下。

先進入後端資料夾：

```powershell
cd backend
```

建立虛擬環境：

```powershell
python -m venv .venv
```

啟動虛擬環境：

```powershell
.venv\Scripts\activate
```

安裝後端套件：

```powershell
pip install -r requirements.txt
```

啟動 Flask：

```powershell
flask run
```

或：

```powershell
python app.py
```

後端常見網址：

```text
http://localhost:5000/
```

## 常見問題

### npm install 失敗

可以先確認 Node.js 是否安裝：

```powershell
node -v
npm -v
```

如果沒有版本號，請先安裝 Node.js LTS。

### npm run dev 顯示 vite 不是指令

通常是還沒安裝套件。

請在 `frontend` 資料夾執行：

```powershell
npm install
```

再執行：

```powershell
npm run dev
```

### localhost:5173 打不開

請確認：

- `npm run dev` 還在執行。
- 終端機有顯示 Local URL。
- 瀏覽器網址是否輸入正確。

### PowerShell 說找不到 npm

代表 Node.js 沒安裝，或安裝後尚未重新開終端機。

可以先關掉 VS Code / Cursor，重新打開後再試一次。

## 建議開發順序

1. 先確認前端可以啟動。
2. 熟悉 mock data 與畫面元件。
3. 再開始建立後端 Flask。
4. 再建立 PostgreSQL 資料庫。
5. 最後把前端 mock data 改成呼叫後端 API。
